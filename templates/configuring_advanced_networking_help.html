Configuring the switch

Hardware Requiements Network Switch

Freezy Arena requires a Cisco Layer 3 switch to run advanced networking. Any Cisco switch made from 1999 to present should function permitting it is layer 3. For example, any switch within the 3500 series, 3700 series, 3800 series, or 9300 series will function correctly.

Software Requirements Cisco IOS

In order to configure the Cisco Switch, you will need an RS232 to USB cable, or an RS232 to RJ45 cable with a serial port or adapter. Once you configure the initial settings, you will then be able to access the switch via a terminal emulator (PuTTY or Tera Term recommended). When correctly configured, you should be able to access the switch via telnet at 10.0.100.3.

Configuring the Switch Cisco IOS CLI

To configure the switch, download the latest switch configuration from github.com/cpapplefamily/freesy-arena. Once downloaded, power up the switch and connect the console cable to the console port on the back of the switch. Open your terminal emulator, and connect to the Serial COM port shown in your device manager. Once connected to the switch, wait for it to show Switch>. Once here, type in enable. It should then show Switch#. From here, enter configure terminal. It should then show switch(config)#. You can then copy and paste the entire switch config text file into the terminal emulator. Once it finishes, enter copy running-config startup-config and press enter twice to confirm. Once completed, connect the FMS to port 7 on the switch. You should then be able to run ping 10.0.100.3 if the FMS is configured correctly.

Connecting the field network Field Setup

The Switch should have devices connected as follows:
Field Radio: port 5.
SCC Trunk (Blue): ports 1 and/or 2
SCC Trunk (Red): ports 3 and/or 4
FMS Computer: port 7
Internet Uplink: port 18
Blue Driver Stations (No SCC): B1 - 12, B2 - 14, B3 - 16
Red Driver Stations (No SCC): B1 - 13, B2 - 15, B3 - 17
General Use (Internet and Full Field Network): ports 19 thru 36
Limited Network Access (Internet and FMS Web Only): ports 37 thru 48

The default password for the switch is 1234Five, with IP 10.0.100.3


Configuring the Field AP

Hardware Requirements Field Access Point

Freezy Arena currently only supports the VividHosting VH-113 (VH-109) radio/access point, which requires a passive PoE injector and 2 ethernet cables to run.

Software Requirements VH-109 .img

Freezy Arena requires VividHosting's VH-109/113 firmare version 1.2.9 or later (VH-109_AP_OFFSEASON_1.2.9-02102025).

Configuring the Field AP VividHosting VH-113

To configure the Field AP for use with the FMS, navigate to VividHosting's website and download the offseason version of the Field AP firmware. Copy the MD5 hash to Notepad. Once downloaded, connect the field AP to passive PoE power. Once the radio has powered on, connect the radio to your PC. From a web browser, connect to the VH-109 at IP 192.168.69.1 (Or 10.0.1.1 if defaulted). Choose to 'Update Firmware' and select the VH-113 file. Enter the MD5 hash from notepad, and begin the upgrade. Once the upgrade has completed, you should be able to access the Field AP at 10.0.100.2 when it is connected to port 5 on the Cisco switch.

Cofiguring the FMS for the field AP Freezy Arena Settings

Navigate to Settings, and select the field tab. Under the access point section, you'll enter the following details about the field AP:
Access Point IP: 10.0.100.2
Password: leave blank


Configuring the Radio Kiosk

Hardware Requirements Laptop PC

To setup the radio kiosk, you'll need a computer that supports virtualization technology. Ensure virtualization is enabled in BIOS.

Software Requirements Operating System and Apps

The VividHosting Radio Kiosk requires Linux or Windows 10 (or later), and Docker Desktop to be installed. You will also need to download the installer from VividHosting. You can find the installer at: https://vividhosting.app.box.com/v/frc-radio-kiosk

Installing the Kiosk Installer / Docker

To install the kiosk, start off by ensuring that Docker Desktop isi nstalled and running. Download and extract the zip file from the link in the software requirements. Once extracted, run the executable as administrator. You'll then select the network interface you wish to connect to radios during programming (Usually Ethernet). Once selected, click 'Configure'. Once the program shows 'Configuration successful for interface: Ethernet', you'll be prompted to run a test. Press 'Check Kiosk Status' to verify the installation. If the prompt returns that the kiosk is successfully running, the installation is completed. Click 'Yes' to open the kiosk.

Setting up the Kiosk VividHosting Radio Kiosk

On the FMS, Open a web browser and navigate to Setup > Team List. Scroll down and click the Generate All WPA Keys button. Then navigate to Report > CSV Data Export > WPA Keys and save the file on a USB drive.

On the radio kiosk, open a web browser and go to http://localhost/admin. Enter the password (if required) supercoolpassword. Here, you can see teams that have been programmed and enable various options. Navigate to http://localhost/upload. Upload the exported CSV file from the FMS. The kiosk is now ready for use.
